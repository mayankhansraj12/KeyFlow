# KeyFlow documentation

This documentation is organized by audience so product guidance stays easy to
find without hiding the engineering and release details expected from an
open-source macOS utility.

## For users

- [Getting started](GETTING_STARTED.md) — install KeyFlow, grant permissions,
  and create the first shortcut.
- [Feature guide](FEATURES.md) — supported shortcuts, gestures, screenshots,
  Sound Bar, and window switching.
- [Privacy](PRIVACY.md) — local data, permissions, diagnostics, and thumbnail
  handling.
- [Support](SUPPORT.md) — troubleshooting and what to include in an issue.
- [Release notes](RELEASE_NOTES_0.1.7.md) — changes and limitations in
  KeyFlow 0.1.7.
- [Update policy](UPDATE_POLICY.md) — how manual updates work.

## For contributors

- [Building and releasing](BUILDING.md) — local app builds, universal DMGs, and
  credentialed release builds.
- [Architecture](ARCHITECTURE.md) — module boundaries, runtime flow, safety,
  and concurrency.
- [Performance](PERFORMANCE.md) — performance budgets and profiling procedure.
- [Implementation status](IMPLEMENTATION_STATUS.md) — completed behavior and
  known limitations.
- [Product roadmap](PRODUCT_PLAN.md) — longer-term product and engineering
  direction.
- [Multitouch compatibility](MULTITOUCH_COMPATIBILITY.md) — experimental raw
  trackpad policy and hardware gate.
